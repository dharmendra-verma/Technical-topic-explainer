02The original DistilBERT — what it was before you touched it

Before fine-tuning, DistilBERT was trained for a completely different task — Masked Language Modeling. It doesn't know anything about "documents" or "classes." It only knows how to fill in blanks.

Pretraining objective: given a sentence like "The [MASK] pump was inspected.", predict that [MASK] should be "centrifugal". Trained on BookCorpus + Wikipedia with ~132B tokens worth of updates. This is how it learned English — grammar, vocabulary, word relationships — without any labels.

📚 Components of the original model

1. Token embeddingsLookup(30522 → 768)
2. Position embeddingsLookup(512 → 768)
3. Embedding LayerNorm + Dropout(B, 512, 768)
4. Transformer block × 6self-attn + FFN
5. MLM head — vocab projectionLinear(768 → 30522)

Step 5 predicts which token fills each masked position. It outputs a probability over all 30,522 vocabulary tokens per position.

🚫 What it cannot do

It cannot classify an entire document. There is no "document → class" machinery.

  • Has no concept of "this is a HAZOP report."
  • Has no output head for 15 classes.
  • The [CLS] hidden state exists but is untrained for classification.
  • Its final linear layer predicts words, not labels.
What it does know: English syntax, semantics, and how words co-occur. That knowledge is stored in the embedding + 6 transformer blocks — which is the part we keep.

What's dropped

~23M params removed

The MLM head's vocab projection (768 × 30,522) is discarded. It was only useful for predicting words.

What's new

~0.6M params added

Pre-classifier (768×768) + classifier (768×15). Initialized randomly — learns from scratch during training.

What's reused

~66M params fine-tuned

All encoder weights start from the pretrained values and get adjusted during training — the English knowledge isn't thrown away, just refined for the DDMS domain.

The insight: we don't teach the model English from scratch. It already knows English. We just add a small new head and let training nudge the whole network toward "this is a P&ID" vs "this is a HAZOP report." The classifier head does the heaviest learning; the encoder does the subtlest adaptation.

04How data flows through every layer

Follow one batch — 16 documents, 512 tokens each — through the entire network. At every layer, the shape and the meaning of the tensor change.

📥 Input batch
raw ints
16 documents tokenized by WordPiece, each padded to 512. These are just integer IDs — the model hasn't seen them yet.
(16, 512) · int64
🔢 Token embeddings
Lookup(30522, 768)
Each integer ID is replaced with a learned 768-dim vector. This is where "words become math." Token 8434 (pump) becomes a dense vector representing "pump-ness."
(16, 512, 768) · float
📍 + Position embeddings
Lookup(512, 768)
A 768-dim vector encoding "this is position 0, this is position 1..." is added elementwise. Attention has no notion of order, so this injects sequence information.
(16, 512, 768) · float
🧹 LayerNorm + Dropout
embedding stabilizer
Normalizes each token vector to zero mean, unit variance across its 768 dimensions. Stabilizes training. Dropout zeroes 10% of units randomly (training only).
(16, 512, 768) · float
🧱 Transformer block 1
self-attn → FFN
Each token looks at every other token via self-attention, then passes through an FFN. Early block = local patterns (phrases, collocations, morphology).
(16, 512, 768) · float
🧱 Transformer block 2
deeper features
Mid-level patterns. Syntactic roles. "Noun phrase boundaries." Token vectors get more context-dependent — the vector for pump in "centrifugal pump" diverges from pump in a financial context.
(16, 512, 768) · float
🧱 Transformer blocks 3–5
semantic composition
Each token accumulates evidence from the whole document. The [CLS] token starts aggregating a "summary" of what the document is about. Fine-tuning pushes these layers toward document-topic features.
(16, 512, 768) · float
🧱 Transformer block 6
final representation
Last encoder layer. After training, the [CLS] row now encodes "this is a Standard Operating Procedure," not just "English text." This is where the biggest encoder adaptation happens during fine-tuning.
(16, 512, 768) · float
✂️ Extract [CLS] hidden state
pooling step
Only the first token's vector — the [CLS] row — is kept. It's now a 768-dim summary of the entire document. Everything else is discarded for classification.
(16, 768) · float
🎛️ Pre-classifier Linear + ReLU
NEW · random init
A fresh Linear(768 → 768) layer followed by ReLU. Added at fine-tuning time. Learns a task-specific transformation of the document summary.
(16, 768) · float
🌧️ Dropout(0.3)
regularizer
Zeroes 30% of the 768 units randomly per example. Prevents the classifier from memorizing the training set.
(16, 768) · float
🎯 Classifier Linear
NEW · 15 logits
Linear(768 → 15). Produces one raw score per document class: PID, SOP, IDS, EQS, ELD, PPS, VAL, CED, MR, IR, PFD, MDS, CS, LD, HAZ. No normalization yet.
(16, 15) · float
📉 CrossEntropyLoss
vs true labels
Softmax(logits) → probabilities. Compare against gold labels — how far off is the model? Loss is the signal gradients chase backward through every layer above.
scalar · float
Total tensor transformation: (16, 512) ints of token IDs → (16, 15) floats of class scores → one scalar loss. Every weight matrix in the network is nudged by the gradient of this scalar.

05Inside a transformer block — the smallest atom

Each of the 6 blocks does this. The magic of transformers is that self-attention lets every token see every other token in one step.

① Multi-head self-attention

Project the (512, 768) tensor three times:
• Q = X · Wq (queries)
• K = X · Wk (keys)
• V = X · Wv (values)

Split into 12 heads of dim 64, compute:
A = softmax(Q · Kᵀ / √64) · V

Each token's new vector = weighted sum of every other token's value, where weights come from Q·K similarity. This is how tokens talk to each other.
➜

② Add & LayerNorm

Residual connection: X = X + Attention(X)

LayerNorm normalizes each token's 768-dim vector to zero mean, unit variance.

Residuals let gradients flow through 6 deep layers without vanishing; LayerNorm keeps activations numerically stable.

③ Feed-forward network (FFN)

Two linear layers with GELU in between:
FFN(X) = GELU(X · W1) · W2

Shapes:
• W1: 768 → 3072 (expand)
• GELU activation
• W2: 3072 → 768 (compress)

The FFN is where most of the capacity lives — about 2/3 of a block's parameters. It's a per-token nonlinear feature mixer.
➜

④ Add & LayerNorm

Another residual + LayerNorm:
X = LayerNorm(X + FFN(X))

Output shape is unchanged: (16, 512, 768). This is what the next block receives.

This "attention → add+norm → ffn → add+norm" pattern repeats 6 times in DistilBERT (12 in full BERT).
What attention actually does to a document: in block 1, pump might mostly look at centrifugal and ##ing. By block 6, [CLS] has learned to pay heavy attention to the domain-specific tokens — HAZOP, P&ID, validation — that signal the document type. Fine-tuning doesn't change the mechanics of attention; it changes what each token pays attention to.

07One training step — what happens in ~20 milliseconds

Each epoch consists of ceil(N / 16) of these steps. This is exactly what trainer.train() executes in the inner loop.

1
Sample 16 random examples from DDMSDataset and collate them into a batch.
input_ids (16, 512) · attention_mask (16, 512) · labels (16,)
CPU → GPU
2
Move batch to GPU memory and enter autocast(fp16) context.
Activations will be stored as FP16, master weights stay FP32.
GPU HBM
3
Forward pass: embedding → 6 transformer blocks → [CLS] → head → logits → cross-entropy.
Result: loss = scalar (e.g. 2.71 early, 0.3 late)
all 66M params
4
Scale the loss by a large constant (GradScaler) so tiny FP16 gradients don't underflow to zero.
scaled_loss = loss × 65536 (typical)
GradScaler
5
Backward pass: PyTorch autograd walks the graph in reverse, computing ∂L/∂w for every trainable parameter.
Gradients accumulate in each parameter's .grad attribute.
autograd
6
Unscale gradients and check for NaN/Inf. If found, skip this step.
This is what makes FP16 training safe.
GradScaler
7
Clip gradient norm to 1.0 to prevent any single batch from destabilizing the weights.
clip_grad_norm_(params, 1.0)
clip
8
AdamW optimizer updates every parameter:
w ← w - lr·(m̂ / (√v̂ + ε)) - lr·decay·w
m̂ and v̂ are running mean/variance of gradients.
66M updates
9
LR scheduler ticks one step forward along the warmup/decay curve.
Step 50: lr = 1.5e-5 · Step 500: lr = 1.8e-5 · Step 2000: lr = 0.4e-5
scheduler
10
Zero all gradients so next step starts fresh. Log loss if step % 10 == 0.
optimizer.zero_grad()
reset
Per-step cost on T4

~20 ms / step

With batch size 16 and 512 seq length. Of that:

  • Forward: ~7 ms
  • Backward: ~10 ms
  • Optimizer + clip: ~2 ms
  • Data transfer: ~1 ms
Per epoch

~60–80 s

For 3000 training examples: 3000/16 ≈ 188 steps × 20 ms + eval + save ≈ 1 minute.

TPU v5e-1

~2× faster

BF16 matmul is ~3× more flops; XLA fuses ops. Larger batch size amortizes more.

08Epoch by epoch — what the network is learning

Training isn't uniform. Different phases reshape different parts of the network. Here's the arc of a typical run.

Epoch 1 opening

What's happening: The classifier head is completely random. Its first task is to stop being chaotic and start aligning its 15 output scores with what the pretrained [CLS] embedding is telling it.

Gradient pattern: enormous gradients at the head (W2 fresh-random), medium gradients in the top transformer blocks, tiny gradients at embeddings. The head is literally learning what each of the 15 classes looks like in [CLS]-space.

Dynamics: loss crashes fast (3.0 → 1.4). Accuracy jumps from ~7% (random guess on 15 classes) to ~70%+.

train_loss: 1.42val_f1: 0.71lr: warming up

Epoch 2 consolidation

What's happening: The head is now roughly aligned. Gradients shift: the top transformer blocks start adapting — their attention patterns subtly tilt toward domain-specific tokens (e.g., HAZOP, P&ID, specification).

Encoder adaptation: bottom blocks still mostly unchanged (they handle general English), but blocks 4–6 begin to reshape. The [CLS] vector gradually becomes a better document-topic summary.

Dynamics: LR just past warmup at peak 2e-5. Loss 1.4 → 0.88. F1 continues climbing.

train_loss: 0.88val_f1: 0.83lr: 2.0e-5 (peak)

Epoch 3 refinement

What's happening: Diminishing returns start. The easy documents (distinctive vocabulary: HAZOP → HAZ class) are already perfectly classified. The model now fights over the confusable pairs — e.g., SOP vs VAL, MDS vs EQS.

Attention patterns crystallize: self-attention heads in block 6 settle into stable document-summary routines. Pre-classifier starts learning "decision boundaries" between similar classes.

Dynamics: LR decays. Loss 0.88 → 0.54. F1 0.83 → 0.89.

train_loss: 0.54val_f1: 0.89lr: 1.5e-5

Epoch 4 peak★ best checkpoint saved

What's happening: The model reaches its best-generalizing state. Train and val are both improving, but val improves more — indicating real learning, not memorization.

Weight state: classifier head is now a ~stable function. Top encoder blocks are tuned. Bottom layers barely moved. Dropout (0.3 on head, 0.1 on attention) does the heavy lifting against overfit.

Dynamics: this checkpoint will become the "best" that load_best_model_at_end restores at the end.

train_loss: 0.38val_f1: 0.93lr: 1.1e-5

Epoch 5 plateau

What's happening: Train loss keeps dropping, but val F1 stops improving — classic signal that the model is starting to memorize rather than generalize. This is overfitting creeping in.

Gradients become noise: most updates now teach the network the idiosyncrasies of individual training documents. The core decision rules are already learned.

train_loss: 0.30val_f1: 0.92patience: 1/3

Epoch 6 overfit

What's happening: Train loss still declining. Val F1 stays at 0.92. Model is definitely memorizing. Dropout keeps it from being catastrophic, but extra epochs at this point mostly hurt.

train_loss: 0.27val_f1: 0.92patience: 2/3

Epoch 7 stopped⏹ early stop

What's happening: Val F1 still hasn't beaten epoch 4's 0.93. Patience counter hits 3. EarlyStoppingCallback signals "stop training."

Final action: load_best_model_at_end=True reloads epoch 4's weights. The model returned from trainer.train() is the peak-F1 version, not the final-epoch version.

train_loss: 0.25val_f1: 0.91patience: 3/3 — STOP
The takeaway: Training isn't "keep going until loss hits 0." It's "find the epoch where the network best generalizes to unseen data." Early stopping + load-best-model is how you find it automatically.

Moderate change

Minimal change

These encode general English — tokenization, syntax, basic semantics. They don't need to change much. Aggressively tuning them would destroy hard-won pretraining knowledge.

Why this shape matters: The low LR (2e-5) is precisely what keeps bottom layers from moving much. Had we used 1e-3 (typical for training from scratch), the pretrained encoder would be destroyed and the model would need 100× more data to recover. Fine-tuning is a delicate touch, not a full re-train.

10Key concepts to remember

Transfer learning — we inherit English knowledge from pretraining and only teach the model about documents.
[CLS] pooling — the first token's final hidden state is the document summary the classifier uses.
LR warmup — first 10% of steps ramp from 0→2e-5 to avoid destroying pretrained weights with a random-init head.
Gradient asymmetry — head changes most, top blocks change some, bottom layers barely change.
Mixed precision — FP16/BF16 activations, FP32 master weights. ~2× faster, same accuracy.
Early stopping + best checkpoint — automatically find the peak-generalization epoch; ignore later overfitting.
Fixed-512 padding on TPU — same shape every batch means XLA compiles once, not per-batch.
Weighted F1 — the real success metric; accuracy lies when classes are imbalanced.
